# src/lib/

## Responsibility
Core utility library providing shared infrastructure for the entire application. Owns data access (question fetching, database client), client-side persistence (localStorage for XP, profiles, practice statistics), UI utility hooks (click-outside, local storage sync, sound), rich-text editor helpers (TipTap), authentication token management (College Board JWT), and Tailwind CSS class merging. These modules are the non-UI foundation consumed by pages, components, and API routes.

## Design
- **Multi-source question fetcher pattern** — `questionFetcher.ts` implements a fallback chain for question retrieval: disclosed questions (`-DC` IDs) come from `saic.collegeboard.org`, regular questions try the internal Neon DB API first (`/api/student-qb/question/{id}`), then fall back to the College Board qbank-api. The fetcher also handles fraction-word translation for SPR (student-produced response) answers via `translateFractionWordsToAnswer` and rationale parsing via `findCorrectChoiceOrAnswerOnIBNQuestion`.
- **Serverless database singleton** — `db.ts` initializes a single `neon()` client from `@neondatabase/serverless` using `DATABASE_URL` or `NEON_DATABASE_URL`, and exports revalidation presets (`REVALIDATE_LONG: 3600`, `REVALIDATE_MEDIUM: 300`).
- **localStorage-based persistence layer** — `userProfile.ts`, `practiceStatistics.ts`, and `exportImport.ts` all follow a consistent get/save pattern with JSON serialization, default-state generation on missing data, migration from legacy keys, graceful error handling, and debug logging. The XP system in `userProfile.ts` uses a transaction-log pattern (`XPTransaction[]`) with a 100-entry cap.
- **External URL resolution** — `getInternalAPITargetURL.ts` and `fetchCbJwtTokenInternal.ts` resolve self-referential API URLs by checking `NEXT_PUBLIC_URL`, Vercel environment variables, and falling back to `localhost:3000`. `fetchCbJwtToken.ts` fetches from the College Board `sucred.catapult-prod` JWT endpoint.
- **React hooks as library utilities** — `useClickOutside.tsx` (mousedown/touchstart listener on a ref), `useLocalStorage.ts` (generic hook with `setValue` and `updateValue` for merging), `playSound.ts` (audio element wrapper), `tiptap-utils.ts` (schema checks, node positioning, URI sanitization, image upload stubs).
- **Class-merge utility** — `utils.ts` exports `cn()` wrapping `clsx` + `tailwind-merge`.

## Flow
1. **Question Retrieval** — `fetchQuestionData(questionId, isMyPractice)` is the primary entry point. For disclosed questions it fetches from `saic.collegeboard.org/disclosed/{id}.json`, parses the response into `API_Response_Question` shape (with answer options rekeyed to `A|B|C|D`), and extracts the correct answer from either the `correct_choice` field or regex-parsed from the rationale. For regular questions it calls the internal API at `/api/student-qb/question/{id}`, and on failure POSTs to `qbank-api.collegeboard.org` with `{ external_id }`. Both `mcq` (multiple choice) and `spr` (student-produced response) branches produce the same `QuestionFetchResult` shape.
2. **User Profile & XP** — `getUserProfile()` reads from localStorage key `"userProfile"`, returns a default if missing. `addXPForCorrectAnswer(questionId, scoreBandRange)` and `reduceXPForIncorrectAnswer(questionId, scoreBandRange)` mutate `totalXP`, `questionsAnswered`, `correctAnswers`/`incorrectAnswers`, and append an `XPTransaction` to `xpHistory`. `getXPStatistics(days)` filters recent transactions for summary stats. `debugUserProfile()` is exposed on `window` for dev console.
3. **Practice Statistics** — `getPracticeStatistics()` reads from `"practiceStatistics"` with auto-migration from legacy `"practiceRushStatistics"`. `addQuestionStatistic(entry)` writes a `QuestionStatistic` keyed by `assessment → primaryClassCd → skillCd → questionId`. `addAnsweredQuestion(...)` appends to `answeredQuestionsDetailed` with difficulty and timestamp. Summary functions (`getSkillSummary`, `getDomainSummary`, `getAssessmentSummary`) aggregate stats hierarchically. `updateSessionXP(sessionId, xpChange)` patches `totalXPReceived` in `"practiceHistory"`.
4. **Export/Import** — `exportLocalStorageData()` snapshots all localStorage keys into an `ExportedData` blob, `downloadExportedData()` triggers a file download, and `importLocalStorageData()` restores keys from a parsed file.
5. **JWT Tokens** — `fetchCbJwtToken()` (server-side, calls College Board) vs `fetchCbJwtTokenInternal()` (calls internal `/api/credentials` endpoint), both returning `{ cbJwtToken, status, error? }`.
6. **UI Hooks** — `useClickOutside(ref, callback)` attaches global event listeners. `useLocalStorage(key, initialValue)` returns `[value, setValue, updateValue]` with overlapping-update mitigation. `playSound(soundFile)` plays `/assets/audio/{file}`.

## Integration
- **Consumed by**: `src/app/api/` routes (question fetching, credentials), `src/app/` page components (Practice Rush, Question Bank, Vocabulary), `src/components/` UI components (question display, profile panels, statistics dashboards, TipTap editors)
- **Depends on**: `@/types/question` (API_Response_Question, ExternalID_ResponseQuestion, PlainQuestionType, QuestionById_Data), `@/types/statistics` (PracticeStatistics, AssessmentType, QuestionStatistic, StatisticEntry, AnsweredQuestion, AssessmentSummary, etc.), `@/types/userProfile` (UserProfileWithHistory, XPTransaction, calculateLevel), `@/types/lookup` (DomainItems, SkillCd_Variants), `@/types/session` (PracticeSession), `@neondatabase/serverless` (neon client), `clsx` + `tailwind-merge` (cn utility), `@tiptap/pm/model` and `@tiptap/pm/state` and `@tiptap/react` (editor utilities), `@/components/ui/calendar` (RangeValue type), `fs`/`path` (implied in token fetch usage), environment variables (`DATABASE_URL`, `NEON_DATABASE_URL`, `NEXT_PUBLIC_URL`, `VERCEL_BRANCH_URL`, `CB_MYPRACTICE_SESSION_ID`, `AUTHENTICATION_CB_MYPRACTICE`)
