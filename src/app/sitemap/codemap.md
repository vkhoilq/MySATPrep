# src/app/sitemap/

## Responsibility
Route for an additional XML sitemap served at `/sitemap` (note: this is a page route, not the standard `/sitemap.xml` file generated by `src/app/sitemap.ts`). Provides a static `MetadataRoute.Sitemap` export that lists the same pages as the root-level sitemap but served from a different URL path.

## Design
- **Server-Rendered Sitemap Function** (`sitemap.ts`): Exports a `default function sitemap()` returning `MetadataRoute.Sitemap`. This is Next.js's convention for generating `sitemap.xml` files from App Router route segments.
- Lists the same entries as `src/app/sitemap.ts`:
  - Main pages: home (`/`, priority 1.0), questionbank (0.9), practice (0.9), resources (0.7), review (0.8), question (0.8).
  - Dashboard pages: dashboard (0.8), vocabs (0.8), vocabs/learn (0.8), vocabs/practice (0.8), tracker (0.8), answered (0.7), bookmarks (0.7), sessions (0.7).
- All URLs use `https://www.mysatprep.fun` as the base domain.
- `lastModified` is set to `new Date()` at build/request time for each entry.
- **Duplicate of Root Sitemap**: This file serves as a secondary sitemap endpoint (`/sitemap/sitemap.xml`). The primary sitemap at `src/app/sitemap.ts` generates the main `/sitemap2.xml` (referenced in `robots.ts`). The purpose of this duplicate is unclear — it may be for legacy compatibility or a different sitemap index strategy.

## Flow
1. Next.js routes requests matching `/sitemap/sitemap.xml` to this file.
2. The function builds and returns the `MetadataRoute.Sitemap` array.
3. Next.js serializes it to XML and serves with `Content-Type: application/xml`.

## Integration
- Consumed by: Search engine crawlers that discover this sitemap URL. Not referenced in `robots.ts` (which points to `/sitemap2.xml`).
- Depends on: `next` (MetadataRoute type).
